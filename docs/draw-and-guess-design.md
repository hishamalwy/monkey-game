# خمن وارسم — وثيقة تصميم اللعبة
## مود Gartic.io داخل منصة كلكس

---

## ١. نظرة عامة

**خمن وارسم** هو مود تنافسي يلعب فيه لاعب واحد في كل جولة دور "الرسام"، بينما يتسابق باقي اللاعبين على تخمين الكلمة من خلال الرسم. اللاعب الأسرع في التخمين يحصل على أكبر نقطة، والرسام يكسب نقاطاً بعدد من خمّن. الفائز هو من يصل أول لحد النقاط المحدد.

**الفرق عن "القرد بيتكلم":** في القرد، الجميع يشارك في نفس الوقت حرفاً بحرف. في خمن وارسم، الأدوار تتناوب — رسام واحد، ومجموعة مخمنين.

## ٢. إعداد الغرفة (OnlineSetupScreen)

عند اختيار مود "خمن وارسم" تظهر الخيارات التالية بدلاً من خيارات القرد:

### ٢.١ إعدادات المتاحة

| الإعداد | الخيارات | الافتراضي |
|---|---|---|
| الفئة | نفس قائمة `appCategories` | عام |
| عدد اللاعبين | 3 — 8 | 5 |
| حد النقاط للفوز | 20 / 40 / 60 / 100 | 40 |
| وقت الرسم لكل جولة | 60 ث / 90 ث / 120 ث | 80 ث |
| عدد خيارات الكلمة | 2 / 3 | 3 |

### ٢.٢ بنية `room` في Firestore

```js
{
  mode: 'draw',              // تمييز المود
  category: 'general',
  scoreTarget: 40,
  drawTime: 80,              // ثانية
  wordChoices: 3,
  maxPlayers: 5,
  status: 'waiting' | 'playing' | 'finished',
  hostUid: '...',
  playerOrder: ['uid1', 'uid2', ...],
  players: {
    uid1: { username, avatarId, score: 0, isReady: false }
  },

  // حقول تُضاف عند بدء الجولة:
  currentRound: 1,
  totalRounds: null,         // = playerOrder.length (كل لاعب يرسم مرة)
  drawerUid: 'uid1',
  wordOptions: ['كلب', 'سيارة', 'قمر'],  // مرئية للرسام فقط
  chosenWord: null,          // تُكتب بعد اختيار الرسام — مخفية عن المخمنين
  wordLength: 3,             // طول الكلمة — مرئي للجميع
  roundStatus: 'choosing' | 'drawing' | 'reveal',
  roundStartedAt: timestamp,
  canvasData: '',            // base64 أو stroke array — يُحدَّث مستمراً
  guesses: {
    uid2: { correct: true,  guessedAt: timestamp, points: 18 },
    uid3: { correct: false, guessedAt: null,       points: 0  },
  },
  hint: '_ _ _',            // شرطات تظهر تدريجياً
```

## ٣. تسلسل الجولات

### ٣.١ دورة حياة الجولة

```
[بداية اللعبة]
      ↓
[تعيين الرسام: drawerUid = playerOrder[currentRound - 1]]
      ↓
[roundStatus: 'choosing' — الرسام يختار من 3 كلمات (10 ثانية)]
      ↓
[roundStatus: 'drawing' — drawTime ثانية]
  ┌── الرسام: يرسم على Canvas
  ├── المخمنون: يكتبون تخمينات في Chat
  └── Timer يعد تنازلياً — hint يُكشف تدريجياً
      ↓
[انتهاء الوقت أو تخمين الجميع الصحيح]
      ↓
[roundStatus: 'reveal' — 4 ثواني: الكلمة + نقاط كل لاعب]
      ↓
[currentRound++]
      ↓ (إذا لم يصل أحد لحد النقاط ولم تنته الجولات)
[تكرار من تعيين الرسام]
      ↓
[status: 'finished' — GameOverScreen]
```

### ٣.٢ عدد الجولات

- `totalRounds = playerOrder.length` (كل لاعب يرسم مرة في الراوند الأول)
- إذا لم يصل أحد لـ`scoreTarget` بعد الدورة الأولى، تبدأ دورة ثانية من نفس الترتيب
- اللعبة تنتهي عند أول لاعب يتجاوز `scoreTarget`

## ٤. نظام النقاط

### ٤.١ نقاط المخمنين

نقاط التخمين الصحيح تتناقص مع الوقت:

```js
function calcGuesserPoints(drawTime, guessedAfterSeconds) {
  const ratio = 1 - (guessedAfterSeconds / drawTime);
  const raw   = Math.round(ratio * 100);
  return Math.max(raw, 10); // حد أدنى 10 نقاط للتخمين الصحيح
}
```

مثال بـ drawTime = 80 ثانية:
| التخمين بعد | النقاط |
|---|---|
| 5 ث | ~94 |
| 20 ث | 75 |
| 40 ث | 50 |
| 70 ث | 13 |
| 80 ث (آخر ثانية) | 10 |

### ٤.٢ نقاط الرسام

```js
function calcDrawerPoints(correctGuessersCount, totalGuessers) {
  if (correctGuessersCount === 0) return 0;
  const ratio  = correctGuessersCount / totalGuessers;
  return Math.round(ratio * 60); // حد أقصى 60 نقطة
}
```

مثال: 4 من 4 خمّنوا → 60 نقطة. 2 من 4 → 30 نقطة.

### ٤.٣ عدم الاختيار (Choosing timeout)

إذا لم يختر الرسام كلمة في 10 ثواني → تُختار الكلمة الأولى تلقائياً، ولا يكسب الرسام نقاط في هذه الجولة.

### ٥.٢ شاشة اللعب الرئيسية (`DrawGameScreen`)

**التخطيط العام (من فوق لتحت):**

```
┌─────────────────────────────────┐
│ [R1/4] [اسم الرسام 🎨] [⏱ 45] │  ← Header
├─────────────────────────────────┤
│  _ _ _ _ _  (5 حروف)            │  ← Hint bar
├─────────────────────────────────┤
│                                  │
│         Canvas 300×300           │  ← منطقة الرسم
│                                  │
├─────────────────────────────────┤
│ [🖊][🔲][⭕][↩️]  [🎨 ألوان]    │  ← أدوات (للرسام فقط)
├─────────────────────────────────┤
│ Chat: تخمينات اللاعبين           │  ← صندوق التخمين
│ [____ ادخل تخمينك ____] [إرسال] │
└─────────────────────────────────┘
```

#### ٥.٢.١ Header

- `currentRound / totalRounds` — اليسار
- اسم الرسام الحالي + أيقونة ريشة — المنتصف
- عداد الثواني المتبقية — اليمين (يتحول أحمر عند < 15 ث)

#### ٥.٢.٢ Hint Bar

- شرطات `_ _ _` بعدد حروف الكلمة
- كل 25% من الوقت يُكشف حرف عشوائي:
  - 75% وقت متبقي → حرف أول
  - 50% → حرف ثانٍ
  - 25% → حرف ثالث
- الحروف المكشوفة بالأخضر `var(--bg-green)`, المخفية رمادية

#### ٥.٢.٣ Canvas

**للرسام:**
- `<canvas>` تفاعلي مع pointer events
- أدوات شريط سفلي:
  - قلم (✏️) / مربع (🔲) / دائرة (⭕) / محي (🧹) / تراجع (↩️) / مسح كل (🗑️)
  - لوحة 8 ألوان: أسود, أبيض, أحمر, أزرق, أخضر, أصفر, بني, برتقالي
  - حجم الفرشاة: Thin / Medium / Thick (3 أحجام)
- الرسم يُرسل كـ stroke data عبر Firestore في real-time:
  ```js
  // canvasData: مصفوفة strokes
  [{ tool, color, size, points: [{x,y}, ...] }]
  ```

**للمخمنين:**
- نفس الـ canvas بالقراءة فقط (no pointer events)
- Strokes تُعاد رسمها عند كل تحديث من Firestore
- لا أدوات ظاهرة

#### ٥.٢.٤ Chat / صندوق التخمين

- Messages تظهر بتسلسل زمني (RTL)
- رسائل التخمين الصحيح تظهر بخلفية خضراء: "✅ [اسم] خمّن صح! (+74)"
- الرسائل الخاطئة: تظهر طبيعية بدون الكلمة الحقيقية
- بعد تخمين اللاعب صح: input يتحول لرسائل عادية (لا يعود يؤثر على النقاط)
- الرسام يرى التخمينات لكن لا يستطيع الكتابة في الـ chat أثناء رسمه

**عرض نقاط الجانب:**
- mini scoreboard في الركن العلوي الأيسر (قابل للطي):
  ```
  [🥇] اسم   74
  [🥈] اسم   52
  [🥉] اسم   30
  ```

## ٧. حالات حافة (Edge Cases)

| الحالة | السلوك |
|---|---|
| الرسام يخرج من اللعبة | `drawerUid` يُحذف من `playerOrder`، تنتقل الجولة للتالي |
| لاعب مخمن يخرج | يُحذف من `players`، الجولة تكمل |
| لا أحد خمّن | reveal يظهر "ما خمّن أحد" + الرسام يحصل 0 |
| لاعب واحد فقط | اللعبة تنتهي فوراً مع رسالة "لا يكفي لاعبين" |
| انقطاع الإنترنت | Canvas يحتفظ بآخر حالة محلياً + reconnect يعيد load canvasData |
| كلمات متشابهة | normalize: حذف تشكيل + toLowerCase قبل المقارنة |

## ٩. ملاحظات تقنية للتنفيذ

1. **Canvas sync strategy:** لا ترسل كل pixel، بل أرسل stroke array (tool, color, size, points[]). المخمنون يعيدون رسم كل strokes عند كل تحديث.

2. **Firestore write rate:** تحديث Canvas كل 300ms كحد أدنى (debounce) لتجنب تجاوز Firestore limits.

3. **chosenWord security:** اخزن الكلمة في subcollection منفصلة `rooms/{code}/secret/word` بـ rules تسمح للرسام فقط بالقراءة. المخمنون لا يقرأون `secret`.

4. **Normalized comparison:**
   ```js
   function normalize(str) {
     return str.trim()
       .replace(/[\u0610-\u061A\u064B-\u065F]/g, '') // حذف تشكيل
       .toLowerCase();
   }
   ```

5. **اللغة:** جميع الكلمات عربية. قاعدة البيانات تشمل 200+ كلمة لكل فئة بمستويات صعوبة: سهل (3-4 حروف) / متوسط (5-6) / صعب (7+).
