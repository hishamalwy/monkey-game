# Design Context: Monkey Game (كلكس!)

This project uses a strict **Neobrutalist** design system. All future designs must adhere to these guidelines to ensure consistency and maintain the game's high-energy personality.

## Design Context

### Users
- Groups of friends playing locally or online.
- Competitive players in public rooms looking for quick, high-stakes word games.
- Users want a fun, expressive, and tactile interface that feels like a modern arcade game.

### Brand Personality
- **Energetic**: High contrast colors and bold typography.
- **Playful**: Vibrant palette (Pink, Yellow, Green) and "Monkey" theme.
- **Competitive**: Sharp edges and high-stakes visual feedback (Skull icons, Pulse animations).

### Aesthetic Direction
- **Neobrutalism**: Thick borders (`--brutal-border`), sharp shadows (`--brutal-shadow`), and zero/minimal border-radius.
- **Vibrant Palette**:
    - Primary Action: `--bg-pink` (#FF006E)
    - Background/Identity: `--bg-yellow` (#FFE300)
    - Success/Ready: `--bg-green` (#39FF14)
    - Borders/Text: `--bg-dark-purple` (#2D1B4E)
- **Glassmorphism**: Used sparingly for overlays and floating nav with `backdrop-filter: blur(12px)`.
- **Theme**: Supports both a vibrant Light mode and a deep, luxurious Dark mode.

### Design Principles
1.  **Tactile Brutalism**: Every button must feel "pressable". Use `transform: translate(4px, 4px)` and `box-shadow: none` on `:active` states to simulate physical height.
2.  **High-Energy Motion**: Use the `.pop` (scale) and `.slide-up` (translation) animations for all new elements to keep the UI feeling alive. Timers should `pulse` when critical.
3.  **Information Density**: Keep core game elements (word tiles, drawing canvas, question cards) centered and large. Use "floating islands" for navigation and secondary controls.
4.  **Visual Unification**: Ensure all screens (Monkey, Draw, Survival) share the same card and button styles. Don't use standard browser defaults; everything must be "Brutal".
