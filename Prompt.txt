You are acting as a senior game designer, real-time multiplayer architect, and product engineer working on the KLAX platform.

Your task is to design and integrate a NEW game mode into KLAX as the **5th game**, based on a **host-driven, real-time buzzer game**.

---

# GAME CORE CONCEPT

This is a **real-time, multiplayer, host-controlled party game**.

The game is:

* voice-based
* reaction-based
* category-driven
* host-controlled
* judgment-based

The system does NOT validate answers automatically.
The Host (Judge) is fully responsible for controlling gameplay.

---

# MAIN IDEA

The host selects a category and controls how the question is presented.

There are TWO types of rounds:

---

## TYPE 1 — VISUAL ROUND (Example: Flags)

Flow:

1. Host selects category (e.g. "Flags / أعلام")
2. A visual item is prepared (e.g. flag image)
3. The host clicks "Reveal"
4. The image is shown to all players
5. Players race to press the buzzer
6. First player answers verbally
7. Host judges

---

## TYPE 2 — VOICE ROUND (Example: Movies)

Flow:

1. Host selects category (e.g. "Movies / أفلام")
2. A question is generated for the host ONLY
3. The question is NOT shown to players
4. The host reads the question aloud
5. Players race to press the buzzer
6. First player answers verbally
7. Host judges

---

# HOST CONTROL PANEL (CRITICAL)

The host must have full control over the round.

## Category Selection

* A horizontal list of categories:

  * Flags
  * Movies
  * Songs
  * General Knowledge
  * etc.

---

## Question Control

For each category:

### 1. Change Question

* Button: "Change"
* Loads a new question within the SAME category
* Must be instant

### 2. Reveal (for visual rounds)

* Button: "Reveal"
* Shows content to all players

### 3. Start Round

* Enables buzzer for players

### 4. Judge Controls

* Correct
* Wrong
* Add Time
* Skip / Cancel

---

# PLAYER EXPERIENCE

## Player Screen

Players see:

* Large central **buzzer button**
* Timer
* Category indicator
* Status messages:

  * "Press now!"
  * "You pressed first"
  * "Another player pressed"

For visual rounds:

* they see the revealed image

For voice rounds:

* they only hear the host

# REAL-TIME BUZZER SYSTEM (CRITICAL)

You MUST implement:

* server-authoritative buzzer system
* atomic "first press wins"
* all other players instantly locked
* real-time broadcast to all players

No client-side trust allowed.

# EDGE CASES

Handle:

* host presses change repeatedly
* host changes question after reveal
* player disconnects after winning buzzer
* no one presses buzzer
* two players press at same time
* host disconnects
* rapid spam clicking
